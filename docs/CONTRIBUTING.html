Contributing to slack-events¶

slack-events keeps plugins in the events.callables module. If you’d like to contribute an event source of your own you can submit a pull request to this repository adding a callable object to that module. Events callables MUST conform to the following specification:

  1. Be decorated with the @events decorator from events.utils.
  2. Take no arguments.
  3. Return an events string and a channel string as a tuple.
  • events and channel can be the empty string.
  • If channel is an empty string, then it deaults to SLACK_EVENTS_CHANNEL if set or general.

An example event function might look like this:

@events
def wwucs_ping_pong():
    event = 'Table Tennis Tournament in CF405 @ 5PM!'
    channel = 'pingpong'
    return event, channel

An example events class might look like this:

@events
class WesternWashingtonUniversityComputerScienceTableTenis:
    def __init__(self):
        self.event = 'Table Tennis Tounournament in CF405 @ 5PM!'
        self.channel = 'pingpong'

    def __call__(self):
        return self.event, self.channel

Note¶

The @event decorator sets a special attribute, __event__, on the event function to signal that it is a event, but also for naming purposes. The __event__ attribute is used to provide source for links. For example, the above function would show up in a Slack message like

WWUCS Ping Pong: Table Tennis Tounournament in CF405 @ 5PM!

and the above class would show up like

Western Washington University Computer Science Table Tennis: Table Tennis Tounournament in CF405 @ 5PM!

Therefore, it behooves contributors to pick descriptive names for new event callables.
